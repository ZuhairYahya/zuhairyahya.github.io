Setup MongoDB Sharding

Mon, Dec 13, 2021 3-minute read

Pada artikel ini saya akan menjelaskan mengenai cara melakukan sharding pada MongoDB

Install MongoDB

Untuk Proses install MongoDB, anda dapat melihat tutorial pada artikel INSTALL MONGODB USING VAGRANT

Set Up MongoDB Authentication

  1. Create admin
mongo
  1. connect admin
use admin
  1. create administrative user with root
db.createUser({user: "mongo-admin", pwd: "password", roles:[{role: "root", db: "admin"}]})
  1. exit mongo
exit

Initialize Config Servers

  1. configure mongod.conf
sudo vi /etc/mongod.conf

# mongod.conf

# for documentation of all options, see:
#   http://docs.mongodb.org/manual/reference/configuration-options/

# Where and how to store data.
storage:
  dbPath: /var/lib/mongodb
  journal:
    enabled: true
#  engine:
#  wiredTiger:

# where to write logging data.
systemLog:
  destination: file
  logAppend: true
  path: /var/log/mongodb/mongod.log

# network interfaces
net:
  port: 27017
  bindIp: mongodb

# how the process runs
processManagement:
  timeZoneInfo: /usr/share/zoneinfo

#security:

#operationProfiling:

replication:
    replSetName: configReplSet

sharding:
    clusterRole: "configsvr"

## Enterprise-Only Options:

#auditLog:

#snmp:
  1. restart mongo service
sudo systemctl restart mongod.service
  1. Connect mongoDB to port 27017 with administrative user
mongo mongodb:27017 -u mongo-admin -p --authenticationDatabase admin
  1. initiate replicaset
rs.initiate( { _id: "configReplSet", configsvr: true, members: [ { _id: 0, host: "mongodb:27017" } ] } )
  1. check status replicaset
rs.status()

Configure Query Router

  1. Create a new configuration file
sudo vi /etc/mongos.conf

# mongos.conf

# for documentation of all options, see:
#   http://docs.mongodb.org/manual/reference/configuration-options/

# Where and how to store data.
#storage:
#  dbPath: /var/lib/mongodb
#  journal:
#    enabled: true
#  engine:
#  wiredTiger:

# where to write logging data.

  destination: file
  logAppend: true
  path: /var/log/mongodb/mongos.log

net:
  port: 27019
  bindIp: mongodb

# how the process runs
processManagement:
  timeZoneInfo: /usr/share/zoneinfo

#security:

#operationProfiling:

sharding:
    configDB: configReplSet/mongodb:27017

#snmp:
  1. Create a new systemd unit file
sudo vi /lib/systemd/system/mongos.service

[Unit]
Description=MongoDB Cluster Router
Documentation=https://docs.mongodb.org/manual
After=network-online.target
Wants=network-online.target

[Service]
User=mongodb
Group=mongodb
#EnvironmentFile=-/etc/default/mongos
ExecStart=/usr/bin/mongos --config /etc/mongos.conf
PIDFile=/var/run/mongodb/mongos.pid
# file size
LimitFSIZE=infinity
# cpu time
LimitCPU=infinity
# virtual memory size
LimitAS=infinity
# open files
LimitNOFILE=64000
# processes/threads
LimitNPROC=64000
# locked memory
LimitMEMLOCK=infinity
# total threads (user+kernel)
TasksMax=infinity
TasksAccounting=false

# Recommended limits for mongos as specified in
# https://docs.mongodb.com/manual/reference/ulimit/#recommended-ulimit-settings

[Install]
WantedBy=multi-user.target
  1. start mongos service
sudo systemctl start mongos.service
  1. enable mongos service
sudo systemctl enable mongos.service
  1. check status mongos service
sudo systemctl status mongos.service

Add sharding

  1. create new configuration file on shard1
sudo vi /etc/mongod.conf

# mongod.conf

# for documentation of all options, see:
#   http://docs.mongodb.org/manual/reference/configuration-options/

# Where and how to store data.
storage:
  dbPath: /var/lib/mongodb
  journal:
    enabled: true
#  engine:
#  wiredTiger:

# where to write logging data.
systemLog:
  destination: file
  logAppend: true
  path: /var/log/mongodb/mongod.log

# network interfaces
net:
  port: 27017
  bindIp: mongoshard1


# how the process runs
processManagement:
  timeZoneInfo: /usr/share/zoneinfo

#security:

#operationProfiling:

  replSetName: shardReplSet

sharding:
    clusterRole: "shardsvr"

  1. restart mongodb service
sudo systemctl restart mongod.service
  1. login database sharding
mongo mongoshard1:27017 -u mongo-admin -p --authenticationDatabase admin
  1. Initiate replicaset for sharding
rs.initiate( { _id: "shardReplSet", members: [ { _id: 0, host: "mongoshard1:27017" },{ _id: 1, host: "mongoshard2:27017" }]})
  1. login database mongos
mongo mongodb:27019 -u mongo-admin -p --authenticationDatabase admin
  1. Add sharding
sh.addShard("shardReplSet/mongoshard1:27017,mongoshard2:27017")
  1. Cek status sharding
sh.status()
